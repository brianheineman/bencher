A maneira mais fácil de implantar o [Bencher Self-Hosted][bencher self hosted] é usando um tríptico de subcomandos do `bencher` CLI:

- [`bencher up`][bencher up]: Inicie os contêineres do Bencher Self-Hosted.
  Isto é semelhante a executar [`docker compose up`][docker compose up].
- [`bencher logs`][bencher logs]: Veja a saída dos contêineres do Bencher Self-Hosted,
  se `bencher up` foi executado com [a flag `--detach`][detach flag].
  Isto é semelhante a executar [`docker compose logs`][docker compose logs].
- [`bencher down`][bencher down]: Pare e remova os contêineres do Bencher Self-Hosted,
  se `bencher up` foi executado com [a flag `--detach`][detach flag].
  Isto é semelhante a executar [`docker compose down`][docker compose down].
