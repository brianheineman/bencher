La façon la plus simple de déployer [Bencher en auto-hébergement][bencher self hosted] est d'utiliser un triptyque de sous-commandes du CLI `bencher` :

- [`bencher up`][bencher up] : Démarrer les conteneurs Bencher en auto-hébergement.
  Cela est similaire à l'exécution de [`docker compose up`][docker compose up].
- [`bencher logs`][bencher logs] : Voir la sortie des conteneurs Bencher en auto-hébergement,
  si `bencher up` a été exécuté avec [l'option `--detach`][detach flag].
  Cela est similaire à l'exécution de [`docker compose logs`][docker compose logs].
- [`bencher down`][bencher down] : Arrêter et supprimer les conteneurs Bencher en auto-hébergement,
  si `bencher up` a été exécuté avec [l'option `--detach`][detach flag].
  Cela est similaire à l'exécution de [`docker compose down`][docker compose down].
