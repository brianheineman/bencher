Les tailles binaires peuvent devenir surchargées avec le temps, surtout en {props.lang}. [Le benchmarking continu][continuous benchmarking] est la pratique consistant à exécuter des benchmarks à chaque ensemble de modifications pour s'assurer que les changements n'introduisent pas de régressions de performance. Vous pouvez traiter les tailles binaires comme un benchmark et les suivre dans le CI pour détecter les grandes tailles binaires. La façon la plus simple de mettre en œuvre le benchmarking continu pour les tailles binaires {props.lang} est d'utiliser Bencher.
